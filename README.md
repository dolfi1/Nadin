# Nadin Web App

Проект развивает веб-сервис **«Карточки компаний/участников»**.
Сервис принимает только название компании, нормализует его, ищет карточку в базе и при отсутствии предлагает два сценария создания: автосбор (с обязательным экраном подтверждения) или ручной мастер.

## Что умеет MVP
- Страница поиска `/`: ввод названия компании, показ нормализованного названия, точных и похожих совпадений.
- Переход на карточку при точном совпадении.
- При отсутствии карточки: выбор **Автосбор** или **Создать вручную**.
- Автосбор всегда формирует только черновик и требует подтверждения пользователем перед сохранением.
- Перед сохранением карточки проверяются обязательные поля: `surname_ru`, `name_ru`, `gender`, `ru_org`, `en_org`, `ru_position`, `en_position`.
- Если обязательные поля не заполнены, пользователь автоматически перенаправляется в `/create/manual` с предзаполненными данными и сообщением об ошибке.
- Мульти-источниковый автосбор с приоритетами и фолбэками (ФНС → вторичные реестры → международные источники), с кэшированием ответов провайдеров.
- Ручной мастер `/create/manual` с валидацией.
- Карточка `/card/<id>` с экспортом CSV и аудит-логом изменений.

## Правила, сохранённые при миграции
- Нормализация юрформы и названия RU (ОПФ в конце).
- ОПФ-словарь RU/EN, включая фикс **ПАО ↔ PJSC**.
- Транслитерация по паспортной логике для автогенерации Organization EN.
- Статусы: `Найдено`, `Черновик / Нужно дополнить`, `Нужно проверить`, `Ошибка формата`.

## Запуск
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python web_app.py
```

Приложение стартует на `http://localhost:8000`.

## Тесты
```bash
pytest -q
```

## Ограничения по сбору данных
- Приложение работает в режиме **strict scraping mode** для доменов с антибот-защитой (`rusprofile.ru`): отключены прокси и любые попытки обхода CAPTCHA/anti-bot.
- Рекомендуется использовать легальные источники и официальные API, соблюдать robots.txt, ToS и лимиты запросов.
